Want to choose between Codex, Gemini, Claude, Zhipu/GLM, DeepSeek, Cursor, or another terminal agent? Use tokenomist init-agents, edit the generated commands, then run tokenomist route job.md --agents agents.json. For stricter coding benchmarks, harness/run_agent.py creates pytest-backed logs with objective pass/fail.
How should my logs look? (exact upload format)

One JSON file = one agent's run of one task. To compare agents on the same task, give every file the same task_id and upload them together (multi-select or drag several in). The format is auto-detected from the JSON shape — you don't pick it.

1. Top-level fields (you provide these — they are the "answer key")

FieldTypeRequiredMeaning
agentstringrecommendedDisplay name in the table/race (e.g. "Claude").
modelstringrecommendedDrives cost & latency. Use a known id (see list below).
task_idstringrecommendedSame value across files = same task being compared.
success_turnint or nullrequired*0-based index into the turns array of the first turn with a useful/correct answer. null = never solved.
final_correctboolrequired*Was the final result actually right.
final_score0.0–1.0required*Quality of the final answer. 1.0 = perfect, 0 = failed.

*Without these the run can't be scored: it shows as “failed”, efficiency 0, and the car won't finish the race. They are how the tool knows who actually solved the task.

2. Per-turn fields

FieldTypeMeaning
role / typestringuser, assistant, tool, system (LangGraph: human/ai/tool).
contentstringThe message text.
input_tokens, output_tokensintOptional. If omitted, estimated from text (~4 chars/token).
tool_callsarrayOptional. [{"name","arguments","ok","result"}].
is_correctionboolOptional. Marks a user turn that corrected the agent. Auto-detected from phrasing if omitted.
is_retryboolOptional. Marks an assistant turn that retried. Auto-detected if omitted.

3. The four accepted shapes

  • Native (recommended) — has a "turns" array.
  • OpenAI / ChatGPT — has "messages", each with a "role".
  • Gemini — has "contents", each with "role" + "parts".
  • LangGraph — has "messages", each with a "type" (no "role").

4. Minimal copy-paste example (native format)

{
  "agent": "Claude",
  "model": "claude-sonnet",
  "task_id": "fix-login-bug",
  "success_turn": 2,
  "final_correct": true,
  "final_score": 1.0,
  "turns": [
    { "role": "user", "content": "The login form throws on empty email. Fix it." },
    { "role": "assistant", "content": "Reading the handler.",
      "tool_calls": [ { "name": "read_file", "arguments": { "path": "login.js" }, "ok": true } ] },
    { "role": "assistant", "content": "Added a guard for empty email; tests pass." }
  ]
}

Here success_turn: 2 points at the third element (index 2) — the assistant turn that solved it. Known model ids use family-prefix matching, so dated ids still resolve. Examples: claude-opus-4-8, claude-sonnet-5, claude-haiku-4-5, gpt-5.4, gpt-5.4-mini, gpt-4o, gemini-3.1-pro, glm-5.1, deepseek-v4-pro. Unknown model costs show as n/a instead of using a fabricated rate.
